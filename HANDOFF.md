## Boundary notes (still in force)

- Do not import any code from `reference/just-bash/` if you vendor it.
- No `os/exec` in runtime code (lint rule lands Phase 21).
- No real LLM / network / exec in tests. `make ci` sets `GOBASH_TEST_NO_NETWORK=1`.
- The `interp.(*Runner).call` frame-name match is load-bearing for `MaxCallDepth`.
- mvdan/sh's `syntax.Walk` panics on `*syntax.BraceExp`. Alias / procinfo / transform plugin passes all run BEFORE `enforceExpansionCaps`.
- mvdan/sh's process-substitution goroutine shares CallHandler — per-Exec counters MUST be atomic.
- mvdan/sh auto-fills `$HOME` from the host user when env doesn't supply one.
- **mvdan/sh hardcodes `$$` to `os.Getpid()` and `$PPID` to `os.Getppid()`** — virtualization is at the AST level (`procinfo.go`). Only `$X` / `${X}` is rewritten.
- **`$BASHPID` counter is per lexical `(...)` subshell only.** Background `&`, process substitution `<(cmd)`, and pipeline stages do NOT bump.
- `mvinterp.NewExitStatus` is deprecated. Use `mvinterp.ExitStatus(uint8)`.
- `command.Registry.Register` / `RegisterBuiltin` are NOT concurrent-safe.
- `command.Context` grows phase by phase. Phase 14 added NO new fields (Limits.MaxSqliteTimeout already existed since Phase 2). Phase 15 will add `JSBootstrap` (`InvokeTool` is already type-aliased in `command/types.go`).
- **Mvdan shadow shape (Phase 11):** `export`/`declare`/`local`/`readonly` are NOT command-form builtins — they're `*syntax.DeclClause`.
- **`Bash.Shopt()` defaults to all-off.** Hosts must `b.Shopt().Set("expand_aliases", true)` before Exec to enable alias expansion.
- **`/bin/.` is an invalid filename** — `applyDefaultLayout` skips names equal to `.` / `..` or containing `/` / `\0`.
- **Phase 9 caveats (still in force):** `BashOptions.Fetch != nil` makes `Network` a no-op; `NewSecureFetch(nil)` is DENY-ALL; `MaxRedirects` / `Timeout` / `MaxResponseSize` use 0=default / negative=disable / positive=cap.
- **Phase 10 caveats unchanged** — `curl` runtime nil-check on `c.Fetch`; `-L`/`-k` are no-ops governed by host Doer; `html-to-markdown` ConvertString has no trailing newline; etc.
- **Phase 13 caveats unchanged** — Origin mutation, PIPESTATUS omission, per-call counter, single-stage wrap, append-only plugins, no self-reference cycles.
- **Phase 14 caveats (new):**
  - `sqlite/` subpackage Register **overrides** the Phase 10 Wave H stub via last-writer-wins on `b.Registry().Register`. Call Register exactly once, before Exec.
  - File DBs shuttle through `os.MkdirTemp` + `os.WriteFile` and write back via `c.FS.WriteFile` on cleanup. Concurrent writers race; last cleanup wins.
  - **JSON output uses alphabetical key order** (Go stdlib `encoding/json` map encoder). The TS port emits column-ordered keys; fall back to `-csv` or `json_object()` in SQL if order matters.
  - **CSV uses CRLF** (`csv.Writer.UseCRLF = true`).
  - **Timeout interrupts via `db.Close()`** in a goroutine on `ctx.Done()`. `QueryContext` cancellation alone did NOT interrupt recursive CTEs on the modernc driver.
  - **Stub fixture lives at `internal/testdata/fixtures/sqlite3-stub/`.** The original `sqlite3/` directory is now reserved for real-runtime fixtures consumed by the `sqlite/` subpackage's own comparison-test harness.
  - **`cmpfixture` has no setup hook.** `sqlite/comparison_test.go` inlines its own fixture-runner that calls `Register` before Exec. Phases 15/16 should follow the same pattern (or extend cmpfixture with a `WithSetup` knob — see open follow-ups).
